    # 单链表

    # 1. 两个单链表相交的一系列问题

    问题：给你两个单链表（可能有环，可能无环）的头节点 headA 和 headB ，请你找出并返回两个单链表相交的起始节点。如果两个链表不存在相交节点，返回 null 。

    判断有没有环(Leetcode142环形链表II (opens new window))：

    1. 使用哈希表（额外空间），存入；出现null没有环，出现第一次有即为入环结点。
    2. 快慢指针：如果有环，快慢指针一定会相遇（套圈了）。找到入环的第一个结点方法（快指针回到第一个，接着快慢指针都一次走一步）（先记住流程结论，证明不用）

    讨论无环链表的情况。

    讨论一个为空一个不为空的情况。

    讨论两个不为空的情况（画图，三种）。

    这题综合。

    解法弄清楚

    // 判断链表是否有环

public class Solution {
    public ListNode detectCycle(ListNode head) {
        if(head==null||head.next==null||head.next.next==null) {
            return null;
        }
        /*解法1：哈希表 */
        /*解法2：快慢指针 
        思路：*/
        
        // 正确写法
        ListNode slow = head.next;
        ListNode fast = head.next.next;
        // 错误写法
        /*
        ListNode slow = head;
        ListNode fast = head;       
        */
        
        
        while(slow!=fast) {
            // 在里面判断即可，这种方法可以借鉴
            if(fast.next==null||fast.next.next==null) {
                return null;
            }
            slow = slow.next;
            fast = fast.next.next;
        }
        // 正确的这里应该是fast=slow
        fast = head;
        while(slow!=fast) {
            slow = slow.next;
            fast = fast.next;
        }
        return fast;
    }
}

    # 2. 两数相加

    给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照 逆序 的方式存储的，并且每个节点只能存储 一位 数字。

    请你将两个数相加，并以相同形式返回一个表示和的链表。

    你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

    这题自己陷入了常规思维，应该将这题抽象出来。这里一开始想着得到加和，把这个链表整成整数形式，其实没必要，直接各个位置相加即可。这里要考虑进位相加的写法。

    /**
 * Definition for singly-linked list.
 * public class ListNode {
 *     int val;
 *     ListNode next;
 *     ListNode() {}
 *     ListNode(int val) { this.val = val; }
 *     ListNode(int val, ListNode next) { this.val = val; this.next = next; }
 * }
 */
class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        ArrayList<Integer> array = new ArrayList<>();
        int sum = 0;
        int carry = 0;
        while((l1!=null)&&(l2!=null)) {
            // sum = (l1.val+l2.val)%10+carry;
            // carry = (l1.val+l2.val)/10;
            array.add(((l1.val+l2.val)%10+carry)%10);
            carry = (l1.val+l2.val+carry)/10;
            l1=l1.next;
            l2=l2.next;
        }
        while(l1!=null) {
            array.add(((l1.val+0)%10+carry)%10);
            carry = (l1.val+0+carry)/10;
            l1=l1.next;
        }
        while(l2!=null) {
            array.add(((l2.val+0)%10+carry)%10);
            carry = (l2.val+0+carry)/10;  
            l2=l2.next;         
        }
        if(carry==1) {
            array.add(1);
        }
        ListNode head = new ListNode(array.get(0));
        ListNode pre = head;
        for(int i=1; i<array.size(); i++) {
            pre.next=new ListNode(array.get(i));
            pre=pre.next;
        }
        return head;
    }
}

    }
}

    这一题的想法是：构造大根堆，其中首尾互换k次即可，可以减少堆排序的次数（取中间的一个结果），拿到第k个大的数据。（堆排序法）。我的写法出现了错误，时间超出了限制。

    还有一种思想：建立一个大根堆，做 k - 1k−1 次删除操作后堆顶元素就是我们要找的答案（和我们上一个方法思想一致，中间细节可能不同）。

    class Solution {
    public int findKthLargest(int[] nums, int k) {
        int heapSize = nums.length;
        buildMaxHeap(nums, heapSize);
        for (int i = nums.length - 1; i >= nums.length - k + 1; --i) {
            swap(nums, 0, i);
            --heapSize;
            maxHeapify(nums, 0, heapSize);
        }
        return nums[0];
    }

    // 构造大根堆
    public void buildMaxHeap(int[] a, int heapSize) {
        for (int i = heapSize / 2; i >= 0; --i) {
            maxHeapify(a, i, heapSize);
        } 
    }
    
	// heapify操作
    public void maxHeapify(int[] a, int i, int heapSize) {
        int l = i * 2 + 1, r = i * 2 + 2, largest = i;
        if (l < heapSize && a[l] > a[largest]) {
            largest = l;
        } 
        if (r < heapSize && a[r] > a[largest]) {
            largest = r;
        }
        if (largest != i) {
            swap(a, i, largest);
            maxHeapify(a, largest, heapSize);
        }
    }

    // 交换
    public void swap(int[] a, int i, int j) {
        int temp = a[i];
        a[i] = a[j];
        a[j] = temp;
    }
}

    # 字符串

    # 1. 无重复字符的最长子串 (opens new window)

    题目

    给定一个字符串 s ，请你找出其中不含有重复字符的 最长子串 的长度。

    示例 1:

    输入: s = "abcabcbb" 输出: 3 解释: 因为无重复字符的最长子串是 "abc"，所以其长度为 3。 示例 2:

    输入: s = "bbbbb" 输出: 1 解释: 因为无重复字符的最长子串是 "b"，所以其长度为 1。 示例 3:

    输入: s = "pwwkew" 输出: 3 解释: 因为无重复字符的最长子串是 "wke"，所以其长度为 3。请注意，你的答案必须是 子串 的长度，"pwke" 是一个子序列，不是子串。

    思路

    第一种：

    哈希表+双指针：遍历存值，将遇到的值一个一个和存入的值相比较，相同，则考虑一次max，且做相应处理，要移动指针，这里可能较为复杂。

    这里试过很多次都不能考虑所有情况，参照别人

    第二种：

    哈希表+队列：但是这里不怎么清楚如何确定每个点的位置，借助队列的索引？有吗？这里是通过hashmap的索引得到。

    第三种：

    滑动窗口，很像第二种我想到的队列的方法。注意，这里用到的是它的思想，但是它只是通过指针即可实现，不需要过多的空间。

    想法

    这种题目，一开始可能有点思路，但是不一定简单准确合适。首先需要积累多的武器，其次还需要分析深层次的问题，总结归纳。即这种题目为什么需要这么做？避免走很多弯路。

    代码

    public int lengthOfLongestSubstring(String s) {
        HashMap<Character, Integer> map = new HashMap<>();
        int maxLen = 0;//用于记录最大不重复子串的长度
        int left = 0;//滑动窗口左指针
        for (int i = 0; i < s.length() ; i++)
        {
            /**
            1、首先，判断当前字符是否包含在map中，如果不包含，将该字符添加到map（字符，字符在数组下标）,
             此时没有出现重复的字符，左指针不需要变化。此时不重复子串的长度为：i-left+1，与原来的maxLen比较，取最大值；

            2、如果当前字符 ch 包含在 map中，此时有2类情况：
             1）当前字符包含在当前有效的子段中，如：abca，当我们遍历到第二个a，当前有效最长子段是 abc，我们又遍历到a，
             那么此时更新 left 为 map.get(a)+1=1，当前有效子段更新为 bca；
             2）当前字符不包含在当前最长有效子段中，如：abba，我们先添加a,b进map，此时left=0，我们再添加b，发现map中包含b，
             而且b包含在最长有效子段中，就是1）的情况，我们更新 left=map.get(b)+1=2，此时子段更新为 b，而且map中仍然包含a，map.get(a)=0；
             随后，我们遍历到a，发现a包含在map中，且map.get(a)=0，如果我们像1）一样处理，就会发现 left=map.get(a)+1=1，实际上，left此时
             应该不变，left始终为2，子段变成 ba才对。

             为了处理以上2类情况，我们每次更新left，left=Math.max(left , map.get(ch)+1).
             另外，更新left后，不管原来的 s.charAt(i) 是否在最长子段中，我们都要将 s.charAt(i) 的位置更新为当前的i，
             因此此时新的 s.charAt(i) 已经进入到 当前最长的子段中！
             */
            if(map.containsKey(s.charAt(i)))
            {
                left = Math.max(left , map.get(s.charAt(i))+1); // 注意滑动窗口的这个max比较
            }
            //不管是否更新left，都要更新 s.charAt(i) 的位置！
            map.put(s.charAt(i) , i); // 若不存在，创建；若存在。更新。每次都需要更新
            maxLen = Math.max(maxLen , i-left+1);
        }
        
        return maxLen;
    }
