    # XML是什么

    XML指的是可扩展标记语言。它被设计用来传输和存储数据。它的重点在于数据的内容，不同于HTML语言的重点在于数据的外观。

    XML没有预定义(即可扩展)的标签,这样就允许我们定义自己的标签和自己的文档结构。

    XML 是独立于软件和硬件的信息传输工具。XML 是各种应用程序之间进行数据传输的最常用的工具。

    # XML作用

    1. 用来保存数据，而且这些数据具有自我描述性

    2. (???)作为项目或者模块的配置文件

    3. 可以作为网络传输数据的格式（现在用的不多，主要以JSON为主）

    # XML解析技术介绍

    XML同HTML一样，是标记型文档，都可以使用w3c组织制定的dom技术来解析。

    Document接口表示整个HTML或XML文档。从概念上讲，它是文档的根，并提供对文档数据的基本访问。

    • (过时，了解)DOM解析：dom解析技术是w3c组织指制定的，所有的编程语言都对这个解析技术使用了自己语言的特点进行实现。java也对dom技术解析标记做了实现。
    • (过时，了解)SAX解析：对dom解析技术进行升级。一行一行地读取xml文件进行解析的，不会创建大量的dom对象。内存使用和性能上由于dom解析。
    • 第三方解析：
      • JDOM：dom基础上进行了封装
      • (重点)DOM4j：又对jdom进行了封装
      • pull

    # dom4j解析技术

    dom4j是第三方的解析技术，因此需要使用第三方给我们提供的类库才可以解析xml文件。需要到dom4j官网下载jar包。

    这里演示一下基本使用，后续遇到补充完善。

    • 创建一个lib目录(存放jar包)，并添加dom4j的jar包，并添加到类路径。

    • 读取xml文件，生成document对象：

    • 读取xml文件，生成类 过程：

      1. 读取xml文件获取document文档对象；
      2. 通过document文档对象获取xml根元素;
      3. 通过根元素获取book标签对象；
      4. 遍历，处理每个book标签转换为Book类。

    此部分小结：

    // 读取xml文件
SAXReader reader = new SAXReader();
Document document = reader.read("src/books.xml");
// 通过Document对象获取根元素
Element rootElement = document.getRootElement();
// 通过根元素获取book标签对象
List<Element> books = rootElement.elements("book");
// 遍历，处理每个book标签对象转换为Book类
for (Element book: books) {
    // 获取book对象的各个元素
    Element nameElement = book.element("name");
    String nameText = nameElement.getText();
    String priceText = book.elementText("price");
    String snValue = book.attributeValue("sn");
    // 转换成Book类
    new Book(snValue,nameText,Double.parseDouble(priceText));
}
